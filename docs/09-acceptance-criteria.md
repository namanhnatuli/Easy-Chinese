# Acceptance Criteria

## Auth
- User can sign in with Google
- First login creates a profile automatically
- Admin and normal user see different route access

## Vocabulary admin
- Admin can create a word with required fields
- Admin can add examples
- Admin can publish/unpublish a word

## Lesson browsing
- Public users can browse lesson list
- Public users can open lesson detail pages
- Only published lessons are visible publicly

## Learning
- User can study a lesson in flashcard mode
- User can switch to multiple choice
- User can use typing mode
- Logged-in progress is saved
- Anonymous progress is not stored in database

## Dashboard
- Logged-in user can see counts for:
  - new
  - learning
  - review due
  - mastered
- User can see lesson completion
